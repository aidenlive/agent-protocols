Release Protocol

Audit, harden, and ship in-progress repositories to public-release readiness. Five phases, seven commands, eleven deliverables.

00 release init Detect existing state, archive stale docs, seed RELEASE-STATUS.md. 01 release audit Phase 1. Audit every surface; classify findings; cite evidence. 02 release report {version} Phase 2. Consolidate audit into RELEASE-{version}-REPORT.md with Go/No-Go. 03 release cleanup Phase 3. Remove dead code; standardize naming, config, tooling, CI. 04 release prep Phase 4. Generate deliverables; verify CI, secrets, rollback, observability. 05 release gtm Phase 5. Build launch checklist; assign owners; rehearse rollback. ∞ release sync Reconcile RELEASE-STATUS.md with reality. Run any time.

02

How resolution works

Trigger phrase → command URL → procedure. One fetch, deterministic mapping.

  1. i

    User issues a phrase

    A short, memorable command — release init, release audit, release report 1.0.0.

  2. ii

    Agent reads the manifest

    It fetches /manifest.json and matches the phrase against trigger_phrases.

  3. iii

    Agent fetches the command

    The resolved URL is a markdown file with frontmatter, required steps, rules, and an output contract.

  4. iv

    Agent executes the procedure

    Maintains the state file the protocol declares. Persists progress across sessions.

03

For agents

Three endpoints. That's the contract.

  • GET /.well-known/ai-protocols.json Minimal pointer for autodiscovery.
  • GET /manifest.json Full list of protocols, trigger phrases, and command URLs.
  • GET /{protocol}/{command}.md The procedure itself. Treat as authoritative.

04

Principles

  • i.

    Truth over assumption. Evidence beats narrative.

  • ii.

    Document reality, not intention. If it isn't true today, it doesn't go in the doc.

  • iii.

    Simplicity over fragility. Cleverness costs interest forever.

  • iv.

    Consistency over entropy. A small system rigorously applied beats a big one applied unevenly.

  • v.

    Never assume functionality without verification. Verified means reproduced, not believed.
